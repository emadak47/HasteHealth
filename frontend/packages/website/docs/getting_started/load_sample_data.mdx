# Load Sample Data

## Prerequisites
Before loading sample data, ensure you have the following:
- A running instance of Haste Health (refer to the [Quick Start](/docs/getting_started/quick_start) guide if needed).
- `haste-health` CLI tool installed. You can find installation instructions in the [setting up cli tutorial](/docs/tutorials/cli).

## Download Sample FHIR Data
You can download sample FHIR data files from the Synthea project.
Download R4 data files from [here](https://synthea.mitre.org/downloads).


## Load Sample Data into Haste Health
```bash
for file in ~/my-synthea-data-fhir/*
do
  haste-health api transaction --file $file
done
```

## Verify Data Load
After loading the data, you can verify that the data has been successfully imported by accessing the Haste Health admin interface or by querying the FHIR API directly. 

## Issues
- If you encounter issue related to bundle size (413 http) errors, consider increasing the `MAX_REQUEST_BODY_SIZE` environment variable in your Docker setup.
For example, set `MAX_REQUEST_BODY_SIZE=8388608` for 8mb.
